本地 Coding Agent 运行时实践

围绕 Pico 风格本地 Coding Agent，完成代码阅读、最小 agent loop 复现、工具安全边界、状态持久化、 trace/report 产物和测试基准整理。重点是理解一个能在代码仓库中持续工作的命令行 agent 如何闭环运行。

项目概述

项目来自 `agent项目学习` 工作区。以 Pico 本地 Coding Agent 为主要代码阅读对象，同时实现 `hand-agent` 与 `tiny-agent` 两个教学规模复现，用于拆解 agent 的核心控制循环和安全工具接口。

Agent Loop

  • 用户请求进入 runtime，由 context manager 构造当前 prompt。
  • 模型输出被解析为 tool call 或 final answer。
  • 工具执行层只允许受约束的文件读写与 patch 操作，并校验路径边界。
  • TaskState 记录步骤、工具结果、错误和最终状态。
  • 每次运行输出 trace.jsonl、task_state.json、report.json，便于复盘和评测。

测试与评测

  • 为最小 agent 实现补充 pytest，覆盖 parser、tool、state、run store、CLI 和 agent 限制。
  • 围绕 mini-pico 增加回归测试，避免历史 tool result 被误判为当前轮工具结果。
  • 整理 harness regression、context ablation、working memory、recovery / resume 等评测维度。
  • 敏感配置通过 `.env.example` 说明，不提交真实 API key 或私密环境变量。
