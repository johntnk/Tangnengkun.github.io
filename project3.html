项目描述

为代码仓库里的多轮开发任务写了一个本地 code-agent harness，主要练习模型接入、工具调用、 上下文裁剪、checkpoint / resume、结构化记忆和运行日志。项目重点处理 prompt 变长、 重复读文件、中断后状态恢复、工具调用边界和结果复查这些实际问题。

核心职责与贡献

  1. 本地 Agent Harness 架构： 实现模型客户端、工具执行、会话状态、checkpoint 恢复和运行工件落盘流程； 当前支持 2 类模型后端、7 类工具和 3 类运行工件。
  2. 上下文预算控制： 设计分层上下文管理与预算裁剪机制，在 12 组长上下文配置里，将平均 prompt 长度从 7082 压到 5664， 平均压缩率 16.19%，最高压缩率 33.28%，并保留当前请求所需的信息。
  3. 结构化记忆系统： 将任务摘要、文件摘要、过程笔记和相关记忆召回分开记录，减少多轮任务里反复读同一文件的问题； 在 12 个记忆依赖任务里，follow-up 阶段的重复读文件次数从 60 次降到 0 次。
  4. Checkpoint / resume： 设计 checkpoint / resume 机制，让 agent 在上下文超预算、中断恢复和 workspace 漂移场景下恢复任务状态； 10 个恢复场景全部通过检查，没有出现误信旧状态继续执行的情况。
  5. 工具调用边界： 为工具调用增加参数校验、工作区隔离、敏感操作确认、重复调用拦截、敏感信息脱敏和 partial success 识别； 固定回归任务全部通过，且都在预算内完成并通过 verifier 检查。
  6. 评测拆分与运行记录： 将评测拆成 harness regression、上下文裁剪、记忆收益和恢复正确性几类，分别看运行时接口、模块收益和恢复边界； 产出固定 benchmark、对照实验记录和运行工件汇总，方便后续复查。
