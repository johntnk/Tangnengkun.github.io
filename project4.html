LLM Engineer's Handbook：LLMOps 生产系统与中文适配

基于开源 LLM Engineer's Handbook 代码体系完成一套上线生产系统，覆盖数据采集、特征工程、 指令/偏好数据集生成、RAG 检索服务、SFT/DPO 训练评估和 SageMaker 部署，并补充中文平台与中文模型生态配置。

Production LLMOps / RAG 服务 / SFT-DPO / 中文适配 2026.07 Python / ZenML / Qdrant / FastAPI / SageMaker

项目描述

围绕生产级 LLM 应用，基于开源 LLM Engineer's Handbook 完成线上可用的 LLMOps 系统； 用 ZenML 串联数据采集、清洗切块、embedding 入库、数据集生成、训练评估和部署服务， 以 MongoDB / Qdrant 管理结构化数据与向量检索，并通过 FastAPI 暴露 RAG 服务。 在复现基础上补充中文结构导读、中文内容源入口、DashScope / Qwen 和 bge 系列模型配置， 让系统从数据到服务都有清晰的工程入口。

核心职责与贡献

  1. 生产系统架构落地： 基于 handbook 的 DDD 分层实现，梳理并落地 domain、application、infrastructure、model 四层职责； 将 MongoDB、Qdrant、FastAPI、SageMaker 和 Hugging Face 相关能力接入到统一工程结构中。
  2. ZenML 流水线编排： 用 digital_data_etl、feature_engineering、generate_datasets、end_to_end_data、training、 evaluating 等 pipeline 串起数据和模型生命周期；通过 tools/run.py 和 YAML 配置控制执行入口、参数和缓存策略。
  3. 中文内容源适配： 补充知乎、CSDN、微信公众号等中文平台 ETL 配置，并为知乎、CSDN、微信公众号、微博、小红书 增加 crawler 类和 dispatcher 注册方法；对反爬较重或解析不稳定的平台保留清晰边界，避免影响主链路稳定性。
  4. 中文模型生态配置： 在 settings 和数据集生成逻辑中加入 DashScope / Qwen 配置路径，训练侧使用 Qwen2.5 基座、 ChatML 模板，RAG 侧配置 bge-large-zh-v1.5 embedding 和 bge-reranker-v2-m3 reranker。
  5. 在线 RAG 服务： 完成原始文档查询、清洗、切块、embedding、Qdrant 写入和检索链路； 在线查询侧覆盖 self-query、query expansion、并发向量检索、reranking、FastAPI /rag 接口和 Opik trace metadata。
  6. 训练评估与部署上线： 接通 SFT / DPO 微调、vLLM 批量生成、judge model 评估、SageMaker endpoint 部署/删除/测试入口； 将训练、评估、推理服务和部署操作拆成可复用命令，便于上线后复查和维护。
