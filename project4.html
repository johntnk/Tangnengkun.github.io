LLM Engineer's Handbook：流水线复现与中文适配

基于开源 LLM Engineer's Handbook 代码体系做工程化学习和二次适配，梳理从数据采集、特征工程、 指令/偏好数据集生成、RAG 检索、SFT/DPO 训练到评估和部署的 LLMOps 链路，并补充中文平台与中文模型生态配置。

项目描述

围绕生产级 LLM 应用的常见链路，复现并拆解 LLM Engineer's Handbook 的工程结构； 重点理解数据仓库、向量库、RAG、数据集生成、SFT/DPO、评估和推理服务之间的衔接关系。 在此基础上补充中文结构导读、中文平台配置入口和中文模型生态适配。当前项目定位为工程学习与改造， 不把仍停留在代码入口和配置层面的云端部分写成线上成果。

核心职责与贡献

  1. 项目结构导读与分层梳理： 编写中文结构导读，按 domain、application、infrastructure、model 四层解释核心代码； 梳理 pipelines 与 steps 的关系，帮助从入口、配置、数据流和模型侧实现快速定位代码。
  2. ZenML 流水线理解与入口整理： 梳理 digital_data_etl、feature_engineering、generate_datasets、end_to_end_data、training、 evaluating 等流水线，以及 tools/run.py 中的命令行入口和 YAML 参数注入方式。
  3. 中文内容源适配： 补充知乎、CSDN、微信公众号等中文平台 ETL 配置，并为知乎、CSDN、微信公众号、微博、小红书 增加 crawler 类和 dispatcher 注册方法；对尚未实现的解析逻辑保留 TODO、正文容器和反爬注意事项。
  4. 中文模型生态配置： 在 settings 和数据集生成逻辑中加入 DashScope / Qwen 配置路径，训练侧使用 Qwen2.5 基座、 ChatML 模板，RAG 侧配置 bge-large-zh-v1.5 embedding 和 bge-reranker-v2-m3 reranker。
  5. RAG 与特征工程链路拆解： 梳理原始文档查询、清洗、切块、embedding、Qdrant 写入和检索链路； RAG 查询侧覆盖 self-query、query expansion、并发向量检索、reranking 和 FastAPI /rag 服务入口。
  6. 训练评估与部署边界整理： 阅读并整理 SFT / DPO 微调、vLLM 批量生成、judge model 评估、SageMaker endpoint 部署/删除/测试入口； 页面中仅描述代码入口和配置路径，不声明云端训练或线上服务已经完成。
