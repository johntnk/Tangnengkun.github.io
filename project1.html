Mini-LLaVA 训练与推理框架

从 LLM / MLLM 基础学习、CLIP / BLIP / LLaVA 论文与代码实践出发，整理并实现轻量级 Mini-LLaVA 训练与推理链路，重点关注视觉编码器到语言模型的连接、数据格式、LoRA 微调和工程可测试性。

独立实现 / 工程实践 2026.06 PyTorch / Transformers

项目概述

该项目来自 `From0to1-MLLM-StudyLog-main` 的多模态学习与实践主线。工作内容覆盖环境搭建、 LLM 推理脚本、SFT / LoRA / QLoRA 入门、CLIP 图文检索、BLIP / BLIP-2 caption、以及 Mini-LLaVA 的训练脚本、推理脚本和模块化重构。

架构与流程

  • 视觉侧使用 CLIP ViT 风格编码器提取 patch-level 图像特征。
  • 通过两层 MLP Projector 将视觉特征映射到 LLM embedding 空间。
  • 语言侧使用 Qwen1.5 量级模型作为 decoder，并用 LoRA 控制可训练参数规模。
  • 训练数据以 image + conversation 形式进入 DataLoader，只对 answer 部分计算 loss。
  • 训练与推理参数通过 YAML 配置管理，便于后续切换模型、数据和生成参数。

技术栈

PyTorch Transformers Accelerate PEFT / LoRA CLIP BLIP / BLIP-2 LLaVA YAML pytest

产出与边界

  • 整理 24 周 MLLM 学习路线，覆盖 CLIP、BLIP、BLIP-2、LLaVA、Mini-LLaVA、部署与优化方向。
  • 形成 Mini-LLaVA 模块拆分：vision encoder、LLM decoder、projector、dataset、train、infer。
  • 补充 shape 校验、前向检查和最小测试，降低后续重构时的接口风险。
  • 当前展示为学习型工程实践，不把短训或示例脚本夸大为完整 SOTA 复现。
