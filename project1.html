Mini-LLaVA：轻量多模态训练与推理框架

围绕从基础图文模型到 Mini-LLaVA 原型的学习与工程实践，覆盖 CLIP、BLIP、BLIP-2、LLaVA 和自建 Mini-LLaVA 完成模型结构、数据格式、训练脚本、推理脚本和模块化重构，重点解决视觉特征如何接入语言模型、 多模态指令数据如何组织、训练链路如何可配置和可测试的问题。

模型结构实现 / 训练推理工程 / 测试重构 2026.06 PyTorch / Transformers

项目描述

面向轻量级多模态大模型训练实践，把图文基础模型和 LLaVA 的关键结构拆成可运行的 Mini-LLaVA 原型； 重点实现视觉编码器、Projector、LLM Decoder、LoRA 微调、answer-only loss、配置化训练和推理脚本， 让后续更换模型、数据和任务时有清晰的代码入口。

核心职责与贡献

  1. MLLM 学习路线与工程拆解： 整理从 LLM 环境、SFT / LoRA / QLoRA、CLIP 图文检索、BLIP / BLIP-2 caption 到 LLaVA / Mini-LLaVA 的阶段化路线，把 24 周学习计划落到可读笔记、代码目录和后续项目模块上。
  2. Mini-LLaVA 模型结构实现： 设计并实现 CLIP-ViT 视觉编码器、两层 MLP Projector、Qwen1.5 语言模型 decoder 的连接方式； 通过连续视觉特征映射到 LLM embedding 空间，完成 image token 与文本 token 的拼接输入。
  3. 训练链路与损失设计： 实现 image + conversation 到 token 序列的数据加载与 collator，训练时屏蔽图片和 prompt 部分， 只对 answer 部分计算交叉熵损失；支持 projector 与 LLM LoRA 作为主要可训练参数。
  4. 配置化训练与推理工程： 将模型路径、数据路径、batch size、学习率、checkpoint、生成参数等集中到 YAML 配置； 提供单卡 / Accelerate 启动路径和独立推理脚本，降低后续切换模型和数据集时的改动成本。
  5. 多任务数据扩展： 在后续模块中把 caption、VQA、QA 等任务统一成指令对话格式，补充数据转换工具和多任务采样思路， 为 Mini-LLaVA 从单一 caption 训练扩展到多任务 SFT 做准备。
  6. 代码重构与质量控制： 将早期训练代码拆分为 dataset、vision_encoder、llm_decoder、mini_llava、train、infer 等模块， 并补充 shape 校验、前向检查和最小单元测试，避免重构后接口不一致。
