AsukaChat-1.3B：中文角色聊天模型训练方案

面向 1.3B 参数中文角色聊天模型，设计从 Base Pretraining 到 Natural Chat SFT、Character SFT、 Character DPO 的完整阶段计划。当前项目处于设计冻结候选与数据源评审阶段，尚未下载数据或训练模型。

训练路线 / 数据治理 / 评测协议 2026.06 1.3B 参数 / 3B token 规划

项目描述

面向中文角色聊天模型从零训练，完成 AsukaChat-1.3B 的阶段化训练方案、数据配比、Tokenizer 约束、 SFT / DPO 样本规范、评测协议和安全边界设计；重点解决角色模型容易变成通用助手、训练数据混杂、 评测 prompt 污染训练集、以及未审查数据源直接进入训练的问题。

核心职责与贡献

  1. 训练路线与阶段门禁设计： 负责设计 Base Pretraining -> Natural Chat SFT -> Asuka Character SFT -> Character DPO 的整体路线， 并拆成 Stage 0 到 Stage 10 的阶段计划，明确每一阶段的输入、产物、审批门禁和禁止动作。
  2. 模型目标与训练规模定义： 固定 1.3B 参数、2048 context、3B token 预训练预算和纯中文聊天场景；把项目目标定义为角色一致性和自然聊天， 而不是通用助手、代码模型、数学模型或安全拒答模型。
  3. 预训练数据配比设计： 设计 5 类数据池：中文通用 20%、中文论坛/对话 35%、小说/对白 35%、情绪生活 5%、中文角色互动 5%； 同时移除日文和中日混合训练池，降低语言漂移和目标混杂风险。
  4. 数据源准入与风险审计： 对 CLUECorpus2020、LCCC / CDial-GPT、TVsub Chinese side、CPED、HER-Dataset、ChatHaruhi-54K 等候选源建立 inventory，记录访问方式、许可状态、PII、污染、去重和版权风险；未通过 review 前不允许下载或训练。
  5. Tokenizer 与数据质量规范： 设计 SentencePiece BPE、50K vocab、特殊 token、byte fallback、中文压缩率和对话标点稳定性检查； 同时定义预训练、SFT、DPO 的去重、乱码过滤、模板重复和低质量样本拒绝规则。
  6. SFT / DPO 样本体系设计： 为自然聊天 SFT、角色 SFT 和角色 DPO 设计样本格式、场景分布、chosen / rejected 偏好标准和拒绝类型， 用数据层面控制 assistant tone、客服腔、过度温柔、纯辱骂、AI 自曝和口头禅堆叠。
  7. 评测隔离与报告模板： 将角色一致性、非助手行为、多轮稳定、中文自然度、非重复作为核心评测轴； 建立 evaluation-only prompt、dataset card、tokenizer report、training report 和 eval report 模板，避免训练与评测边界混乱。
